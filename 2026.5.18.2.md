#
How do I migrate contributions from gitea to github?<br>
如何将 Gitea 中的贡献迁移到 GitHub？<br>
https://forum.gitea.com/t/how-do-i-migrate-contributions-from-gitea-to-github/6610/2<br>
Create a new personal access token at GitHub iif you don’t have one already. It’s in GitHub settings, developer settings, personal access tokens (or in URL bar /settings/tokens/new ). I think the permission is repo:public_repo, but I haven’t documented this clearly, and keep it somewhere at hand.<br>
如果您还没有个人访问令牌，请在 GitHub 上创建一个。您可以在 GitHub 设置 > 开发者设置 > 个人访问令牌（或者在地址栏中 /settings/tokens/new ）中找到它。我记得权限是 repo:public_repo ，但我没有明确记录下来，建议您把它保存好。<br>
Create a new repository at GitHub. Don’t initialise the repository though (don’t add LICENSE or README file), I am not sure if it would result to an error later or if Gitea just overwrote it.<br>
在 GitHub 上创建一个新的仓库。但不要初始化该仓库（不要添加 LICENSE 或 README 文件），我不确定这是否会导致后续错误，或者 Gitea 是否会直接覆盖它。<br>
Go to settings of your Gitea repository and scroll to mirrors.<br>
进入 Gitea 仓库的设置，滚动到镜像部分。<br>
Enter the GitHub HTTPS clone address to Git Remote Repository URL<br>
输入 GitHub HTTPS 克隆地址到 Git Remote Repository URL<br>
Open the authorization menu (just below it)<br>
打开授权菜单（就在它下面）<br>
Set username as your GitHub username and the PAT you made at step 1 as the password.<br>
将 username 设置为您的 GitHub 用户名，将您在步骤 1 中创建的 PAT 设置为密码。<br>
Maybe enable “Sync when commits are pushed”, I am not sure if I have seen that before and if you want, adjust the sync interval.<br>
或许可以启用“提交推送时同步”功能，我不确定之前是否见过这个功能，如果需要，可以调整同步间隔。<br>
Press the “Add Push Mirror” button<br>
按下“添加推拉镜”按钮<br>
By default your commits should get pushed to GitHub every 8 hours, but you could also try the Sychnorize now button to see if it works.<br>
默认情况下，您的提交应该每 8 小时推送到 GitHub，但您也可以尝试 Sychnorize now 按钮，看看是否有效。<br>
